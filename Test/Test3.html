HTML / CSS 元素溢出核心知识

元素溢出通常是指子元素内容尺寸超出父元素可见区域，最常见的表现是横向滚动条或内容被截断。HTML/CSS 中的溢出由 `overflow` 属性控制，常见取值有 `visible`、`hidden`、`scroll`、`auto`，并可分别设置 `overflow-x` 和 `overflow-y`。当一个块级容器没有明确宽度或高度时，子内容会根据默认规则继续扩展，导致父容器或页面出现溢出。

常见溢出原因包括：固定宽度加上过长文本、图片或按钮；内边距、边框、外边距未考虑在总宽度内；Flexbox/Grid 布局中的 `min-width:auto` 或 `flex-shrink:0`；以及 `white-space:nowrap`、`overflow-wrap:normal` 等属性使文本不换行。特别是 `display:flex` 时，子元素默认不收缩，容易把父容器撑开。

判断溢出时常用 `scrollWidth`、`clientWidth`、`offsetWidth`：`scrollWidth` 包含内容的实际宽度（包括不可见部分），`clientWidth` 是内容区宽度加内边距但不含滚动条，`offsetWidth` 则包含边框和滚动条。若 `scrollWidth > clientWidth`，说明存在横向溢出。

处理溢出问题的方法通常包括：给容器加 `max-width:100%` 或 `width:100%`；使用 `box-sizing:border-box` 让 padding/border 计入总宽度；对文本使用 `overflow-wrap:break-word` 或 `word-break:break-all`；对图像和表格设置 `max-width:100%; height:auto;`；对弹性布局元素设置 `min-width:0`、`flex-shrink:1`。

另外，要注意溢出滚动条出现的位置：如果父元素本身没有 `overflow` 约束，则子元素溢出会传递到文档流，最终在浏览器页面底部出现全局滚动条；如果父元素设置了 `overflow:auto` 或 `overflow:hidden`，则溢出会在该父元素内部处理。了解这一点可以帮你快速定位是局部容器问题还是整个页面布局问题。

scrollWidth 是什么？

scrollWidth 表示元素内部内容的总宽度， 它包括可见内容 + 不可见溢出的内容
与 clientWidth 不同，clientWidth 只包含可见区域宽度（padding 内部，不含滚动条宽度）
offsetWidth 则包含元素边框和滚动条（如果有）
如果： el.scrollWidth > el.clientWidth 说明该元素有横向溢出。
